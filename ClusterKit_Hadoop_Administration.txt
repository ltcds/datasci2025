** Permanently fix the public hostname: (Recommended)
# bug in ambari-agent when Confirm Hosts only edge.example.com cannot register, แก้ด้วยวิธีการตามลิงค์ต่อไปนี้
https://community.cloudera.com/t5/Community-Articles/Why-ambari-host-might-have-different-public-host-name-and/ta-p/24666

1. Create a file with name : "/var/lib/ambari-agent/public_hostname.sh" then in that file add the following line:
#!/bin/sh
echo `hostname -f`

2. Make sure that the file "/var/lib/ambari-agent/public_hostname.sh" has proper execute permission. Example:
chmod 755 "/var/lib/ambari-agent/public_hostname.sh"

3. On every ambari-agent host edit the file "/etc/ambari-agent/conf/ambari-agent.ini" and in the [agent] section add the following line:
## Added following to customize the public hostname
public_hostname_script=/var/lib/ambari-agent/public_hostname.sh

NOTE: Users can also use the property "hostname_script" to customize the internal hostname.

4. Make sure that the changes are pushed to all the hosts present in the ambari cluster.
pscp /var/lib/ambari-agent/public_hostname.sh
pscp /etc/ambari-agent/conf/ambari-agent.ini

5. Now restart the agents.
ambari-agent restart

6. send to all hosts
pscp /var/lib/ambari-agent/public_hostname.sh
pscp /etc/ambari-agent/conf/ambari-agent.ini
pdsh -w ^hosts ambari-agent restart

<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

    <value>*</value>
